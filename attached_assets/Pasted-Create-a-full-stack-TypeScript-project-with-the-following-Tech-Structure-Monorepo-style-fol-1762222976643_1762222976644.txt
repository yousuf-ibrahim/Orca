Create a full-stack TypeScript project with the following:

## Tech + Structure
- Monorepo style folders:
  /server  (Node 20, Express, TypeScript)
  /web     (React 18 + Vite + TypeScript)
  /shared  (TypeScript types)
- Add ESLint + Prettier (strict) to both /server and /web.
- In Replit “Secrets”, create env vars:
  JWT_SECRET (random string)
  ADMIN_USERNAME (e.g., demo@orca.io)
  ADMIN_PASSWORD_HASH (bcrypt hash of demo password; provide a script to generate)

## Server requirements (/server)
- Dependencies: express, cors, helmet, zod, jsonwebtoken, bcryptjs, cookie-parser, node-cache, morgan
- src/index.ts:
  - Express with helmet, CORS (origin: the Vite dev origin), cookie-parser, morgan.
  - Routes under /api.
- Auth:
  - POST /api/auth/login { username, password } → verify against ADMIN_USERNAME + ADMIN_PASSWORD_HASH.
  - On success, issue HTTP-only cookie orcajwt (JWT signed with JWT_SECRET, 24h expiration) containing { sub, role: 'admin' }.
  - GET /api/auth/me → returns { authenticated, user: { email, role, investorType? } } if valid cookie.
  - POST /api/auth/logout → clear cookie.
- Investor classification:
  - POST /api/investor-classification { investorType: 'professional' | 'accredited' | 'retail' }
  - Store “last saved” per sub in a simple in-memory NodeCache for demo.
  - Middleware requireAuth that blocks access to protected routes if JWT missing/invalid.
- Security:
  - Use SameSite='lax', secure: true when served over HTTPS; in dev allow non-secure.
  - Rate-limit login (basic in-memory counter by IP; lockout after 5 tries for 10 minutes).

## Common Replit pitfalls to avoid
- CORS: cors({ origin: 'http://localhost:5173', credentials: true })
- Cookies: axios withCredentials: true; Express cookies configured per environment.
- Type safety: Use zod to validate POST bodies; return 400 on invalid.
- Error handling: Central errorHandler middleware; hide stack traces in prod.
- Env access: Never log secrets; read via process.env; throw if missing.
